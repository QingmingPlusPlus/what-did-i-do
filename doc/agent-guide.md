# Agent 协作指南

`AGENTS.md` 是根层硬约束；`doc/agent-memory-map.md` 是开发前主动读取项目记忆的入口。

## 三层治理

- `spec` 或 `openspec/specs/`：保存需求约束、验收场景、长期业务规则，以及明确由 spec 派生的设计决策。
- `doc/`：保存模块说明、当前实现状态、页面结构、字段清单、路由、接口现状、开发记录和迁移记录。
- `.agents/skills/`：只保存稳定、重复、值得复用的操作知识；必须具备明确入口、约束、步骤和自检。

不要把页面说明、字段清单、当前接口特例、一次性迁移背景或项目现状放入 skill。

## 读取策略

1. 根据用户请求识别目标路径、模块、页面、接口或 spec 能力。
2. 打开 `doc/agent-memory-map.md`，选择最具体的匹配行。
3. 先读匹配的 `doc` 和 `spec`，再按需使用匹配的 skill。
4. 若没有匹配行，用 `rg` 在 `doc/`、`spec/`、`openspec/specs/`、`.agents/skills/` 中搜索相关模块名、页面名、接口名或路径。
5. 若本次改动新增了长期入口、模块说明、spec 或稳定流程，更新 `doc/agent-memory-map.md`。

## 改后同步

- 新增或改变需求约束：更新对应 spec。
- 新增或改变当前项目状态：更新对应 `doc/`。
- 新增或改变稳定可复用流程：更新或新增 `.agents/skills/`。
- 新增页面、字段、mock、路由、当前接口参数或页面布局时，默认更新 `doc/`，不要新增 skill。

## 保鲜规则

- 多份记忆重叠时，优先使用最近验证过的记录。
- 过期文档或路由行要标记为 deprecated，不要留下互相冲突的 active 指引。
- 当代码和文档不一致时，先检查当前实现，并在本次改动中同步更新过期记忆。
